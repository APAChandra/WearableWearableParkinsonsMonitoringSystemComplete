Detailed Description

A simple WearableListView.Adapter that accepts an array of strings as its data set. Developers can instantiate this adapter and pass a string array to the constructor. The default behavior is to use a filled-in circle for each row. The off-centered rows will be using a faded color for both the text and the circle while the centered row will have a larger circle to stand out. Developers can optionally add an icon to the circles.

A number of resources can be set to change the look and feel of each row; consult the resource file

wcl_list.xml

for details (to customize these values, copy this resource to your client application and change the values as needed).

To be notified when a row is clicked on, implement WearableListView.ClickListener#onClick(WearableListView.ViewHolder): 

  public void onClick(WearableListView.ViewHolder v) {
       Integer position = (Integer) v.itemView.getTag();
       // use this data to complete some action ...
   }

It is also possible to show the "checked" state of an item. To enable this, call

setCheckable(true)

(the default value is

false

).

The same method can also be used to make the rows dynamically selectable; you would need to add the following snippet to the WearableListView.ClickListener#onClick(WearableListView.ViewHolder): 

       public void onClick(WearableListView.ViewHolder v) {
           Integer tag = (Integer) v.itemView.getTag();
           adapter.setChecked(tag, WearableListConfig.NO_SELECTION);
           adapter.notifyDataSetChanged();
           // use this selection to complete some action ...
       }

Sets the state of the row to "checked" at the index

position

and uses the icon

checkedResourceIcon

to visually show that state. In order for this method to be take effect, you also need to make the list "checkable" by calling

setCheckable(true)

. If

WearableListConfig.NO_SELECTION

is passed for the icon, the default resource identified in the

R.drawable.wcl_list_default_checked_icon

alias will be used. For

checkedPosition < 0

or

checkedPosition > n - 1

where

n

is the size of the data array, calling this method will not do anything.
