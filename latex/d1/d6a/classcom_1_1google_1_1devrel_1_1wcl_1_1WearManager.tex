\subsection{Detailed Description}
A singleton to manage the wear interaction between a device and its connected devices. This class maintains certain states of the system and hides direct interaction with the \hyperlink{}{Google\+Api\+Client}. Clients should initialize this singleton as early as possible (e.\+g. in the on\+Create() of the Application instance). Then, accessing this singleton instance is possible throughout the client application by calling \hyperlink{classcom_1_1google_1_1devrel_1_1wcl_1_1WearManager_a5e9a61518e44bb2c1a89706d51240661}{get\+Instance()}.

In most cases, components within the client application need to have access to certain events and receive messages, data items, etc. Each component can register an implementation of the \hyperlink{}{Wear\+Consumer} interface with the \hyperlink{classcom_1_1google_1_1devrel_1_1wcl_1_1WearManager}{Wear\+Manager}. Then, the relevant callbacks within that interface will be called when an event is received or a life-\/cycle change happens. To make this easier, \hyperlink{}{Abstract\+Wear\+Consumer} provides a no-\/op implementation of the \hyperlink{}{Wear\+Consumer} which clients can extend and only override the callbacks that they are interested in.
